# Global Project Context

**Project:** Flow Analysis and Resilience of the Paulista SUS via Graph Theory.

**Main Objective:** Analyze the topology and dynamics of hospital admission flows in the state of Sao Paulo to identify structural vulnerabilities and measure the impact of geographical displacement.

## Technology Stack

- Language: Python 3.10+
- Data: PySUS, Pandas, NumPy
- Graph Modeling: igraph or NetworkX (focus on optimization)

## Domain Model

- Graph Type: Large-scale directed and weighted graph.
- Scale: $|V| > 10,000$.
- Mathematical Notation: $G = (V, E, W)$.
- Nodes ($V$): Municipalities of Sao Paulo and CNES health facilities.
- Edges ($E$): Patient displacement flows (origin -> destination) and transfer links.
- Weights ($W$): Service volume and geographical distance in km.

## Data Sources

- SIH/SUS (RD files)
- CNES
- IBGE API

## Architectural Guidelines

- Separation of Concerns: ETL logic must remain in data-layer functions and must not instantiate graph structures.
- Memory Efficiency: Prefer efficient graph loading and lightweight subgraph/view operations.
- Filter Safety: Filter operations must not mutate the base graph object.
- Priority Algorithms: Support Betweenness centrality and Louvain community detection.
- Error Handling: Use explicit `try-except` blocks for IBGE/API failures and missing DATASUS data; avoid silent failures.
